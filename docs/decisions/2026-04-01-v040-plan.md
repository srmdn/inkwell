# Plan: v0.4.0 Media Library, Site Settings, and Demo

- Date: 2026-04-01
- Status: Accepted
- Owners: @srmdn

## Context

v0.3.0 shipped a complete admin dashboard (post editor, subscribers, rebuild trigger).
Three items were explicitly deferred in the v0.3.0 decision doc:

1. Media library with pluggable storage drivers (local + S3)
2. Site settings editor in the dashboard (site name, description, social links)
3. demo.foliocms.com public demo install

v0.4.0 closes those gaps. After this release, foliocms is demo-ready and posts no longer
require base64-encoded inline images (a known limitation since v0.3.0).

## Goals

1. Give users a media library to upload, browse, and insert images without leaving the admin.
2. Allow inserting images by external URL in the editor (for users who self-host images elsewhere).
3. Persist site-level settings (name, description, social links) in the database, readable by themes.
4. Launch demo.foliocms.com as a public demo install after v0.4.0 ships.

## Decisions

### 1. Pluggable MediaDriver interface

Media storage is abstracted behind a `MediaDriver` interface with two implementations:

- `LocalDriver`: stores files under `content/media/`, served by the backend at `GET /media/*`
- `S3Driver`: uploads to any S3-compatible bucket (AWS S3, Cloudflare R2, MinIO)

Driver is selected via `MEDIA_STORAGE=local|s3` in `.env`. Default is `local`.

**Rationale:** Most self-hosters start with local storage. The interface lets them migrate
to S3 later by changing one env var. The abstraction is minimal: one interface, two
concrete structs, no extra layers.

### 2. Media files tracked in SQLite

Uploaded files are recorded in a new `media` table (key, filename, content_type, size,
created_at). This makes the list endpoint fast without walking the filesystem, and gives
a clean delete path.

**Rationale:** Consistent with the post storage pattern: filesystem holds the data,
SQLite holds the metadata.

### 3. Site settings as a key-value table

A new `settings` table (key TEXT PRIMARY KEY, value TEXT) stores site-level config.
Defaults are seeded on first run.

Exposed via:
- `GET /api/settings` (public) — themes call this to read site name, description, social links
- `GET /api/admin/settings` + `PUT /api/admin/settings` (protected)

**Rationale:** Key-value avoids schema churn when new settings are added. Adding a new
setting is a new row, not a migration column change.

### 4. Two image paths in the editor

The Milkdown editor gains two ways to insert images:

- **Insert by URL**: toolbar button opens a modal for URL + alt text, inserts standard
  Markdown `![alt](url)`. Zero upload required; works with any external image host
  (imgmu.id, CDN, self-hosted).
- **Paste/drop to upload**: pasting or dropping a local image file triggers a background
  upload to `POST /api/admin/media`; the returned URL is inserted as `![alt](url)`.
  Replaces the current base64 DataURI approach.

**Rationale:** Many self-hosters already use an external image host. Forcing them through
the media library is friction. The URL path is the zero-friction option; paste-to-upload
is the convenience path for users who want everything local.

### 5. S3 via AWS SDK v2

The `S3Driver` uses `github.com/aws/aws-sdk-go-v2`. This is the only new external
dependency in v0.4.0. `govulncheck` must pass before any deploy that includes this driver.

### 6. demo.foliocms.com deferred to post-M5

The demo instance will be set up on the VPS after the v0.4.0 binary is built and verified
on `staging.foliocms.com`. Demo mode (read-only, public credentials) will be a runtime
flag, not a separate build.

### M2: Media library — backend

**Interface** (`internal/media/driver.go`):
```go
type MediaDriver interface {
    Upload(filename string, r io.Reader, contentType string) (MediaFile, error)
    List() ([]MediaFile, error)
    Delete(key string) error
    PublicURL(key string) string
}
```

**LocalDriver:** files under `content/media/<uuid>-<filename>`, served at `GET /media/*`

**New env vars:** `SITE_URL`, `MEDIA_STORAGE`

**Migration** `004_media.sql`: `media` table (key, filename, content_type, size, created_at)

**Endpoints:**
- `POST /api/admin/media` — multipart upload
- `GET /api/admin/media` — list
- `DELETE /api/admin/media/{key}` — delete

Files:
- `internal/media/driver.go`
- `internal/handler/media.go`
- `internal/handler/handler.go`
- `internal/config/config.go`
- `migrations/004_media.sql`

## New Env Vars

| Variable | Default | Purpose |
|----------|---------|---------|
| `SITE_URL` | `http://localhost:8090` | Base URL for media file URLs |
| `MEDIA_STORAGE` | `local` | Storage driver: `local` or `s3` |
| `S3_ENDPOINT` | - | S3-compatible endpoint URL |
| `S3_BUCKET` | - | Bucket name |
| `S3_REGION` | `auto` | Region |
| `S3_ACCESS_KEY` | - | Access key ID |
| `S3_SECRET_KEY` | - | Secret access key |

## New Routes

| Method | Path | Auth | Purpose |
|--------|------|------|---------|
| GET | `/api/settings` | Public | Read site settings (for themes) |
| GET | `/api/admin/settings` | JWT | Read site settings |
| PUT | `/api/admin/settings` | JWT+CSRF | Update site settings |
| GET | `/media/*` | Public | Serve local media files |
| POST | `/api/admin/media` | JWT+CSRF | Upload a file |
| GET | `/api/admin/media` | JWT | List uploaded files |
| DELETE | `/api/admin/media/{key}` | JWT+CSRF | Delete a file |

## Out of Scope for v0.4.0

- Multi-user / roles
- Comments
- Plugin system
- Image optimization / resizing
- Media search / tagging
- Video or non-image file types (deferred to a future release)
